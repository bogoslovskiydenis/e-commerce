# Админ панель на Nuxt 3

Админ панель для e-commerce проекта, переписанная с React на Nuxt 3 без TypeScript и CSS.

## Технологии

- Nuxt 3
- Vuetify 3 (UI компоненты)
- Pinia (state management)
- JavaScript (без TypeScript)

## Установка

```bash
cd admin-panel-nuxt
npm install

# Установите модуль Pinia для Nuxt (если еще не установлен)
npm install @pinia/nuxt --save
```

**Важно:** После установки `@pinia/nuxt` добавьте его в `nuxt.config.js`:

```js
export default defineNuxtConfig({
  modules: ['@pinia/nuxt'],
  // ... остальная конфигурация
})
```

Если модуль не установлен, можно использовать плагин `01.pinia.client.js` (уже создан).

## Настройка окружения

Создайте файл `.env` в корне проекта (можно скопировать из `.env.example`):

```bash
cp .env.example .env
```

Отредактируйте `.env` и укажите URL вашего API:

```
API_BASE_URL=http://localhost:3001/api
```

## Запуск

```bash
# Разработка (запустит на http://localhost:3000)
npm run dev

# Сборка для продакшна
npm run build

# Превью продакшн сборки
npm run preview
```

После запуска откройте браузер по адресу: **http://localhost:3000**

Для входа используйте учетные данные администратора из вашей базы данных.

## Структура проекта

```
admin-panel-nuxt/
├── composables/        # Переиспользуемые композаблы
│   ├── useApi.js      # Работа с API
│   └── usePermissions.js  # Система разрешений
├── layouts/            # Layout компоненты
│   ├── default.vue     # Основной layout с меню
│   └── auth.vue        # Layout для страниц авторизации
├── middleware/         # Middleware для роутов
│   └── auth.js        # Проверка авторизации
├── pages/              # Страницы (автоматический роутинг)
│   ├── login.vue       # Страница входа
│   ├── index.vue       # Дашборд
│   └── orders/         # Страницы заказов
├── stores/             # Pinia stores
│   └── auth.js         # Store для аутентификации
└── nuxt.config.js      # Конфигурация Nuxt
```

## Настройка API

В `nuxt.config.js` можно изменить базовый URL API:

```js
runtimeConfig: {
  public: {
    apiBase: process.env.API_BASE_URL || 'http://localhost:3001/api'
  }
}
```

Или через переменную окружения `.env`:

```
API_BASE_URL=http://localhost:3001/api
```
